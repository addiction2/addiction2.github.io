博客搭建--你好世界

2020年9月4日第二次发布

之前的个人博客是租了虚拟主机，在LNMP框架下，利用wordpress搭建的。但是因为一些原因，这次选择了使用GitHub+hexo搭建个人博客（省去了租虚拟主机的钱）。因为搭建过程和之前有很多差别，所以这里整理一下新的搭建流程。

我是根据这篇教程搭建的个人博客：hexo从零开始到搭建完整

  1. 安装Git Bash，NodeJs以及hexo

  2. 搭桥到GitHub

  3. 修改_config.yml文件中的一些配置

    deploy:
    type: git
    repo: https://github.com/YourgithubName/YourgithubName.github.io.git
    branch: master

  4. 执行命令：

    hexo clean
    hexo generate
    hexo server

  5. 上传到github

  6. 绑定个人域名

  7. 修改及配置主题

基础的搭建流程就结束了。我选择的主题是Fluid，看上去还是不错的，具体的主题配置参考配置指南，非常详细，对新手很友好。但是这里想说一下：这个主题的网页打开速度真的是太太太慢了，看了其他人的帖子，应该是主题没有对 js 和 css 进行任何压缩和合并，并且用了很多框架之类的，调用了很多 lib 而且还都是用的本地的，虽然稳定性会更好。因此主题的作者们也提出了一些解决方法，比如选用合适的CDN 来加速 hexo 博客全局访问速度。

因为我的域名之前就进行过工信部和公安备案，所以我选择使用了阿里云的OSS服务并绑定域名，将生成后的 public 目录下全部上传到 OSS，然后可以无服务器部署博客，并且加载速度也将无可比拟。阿里云配置CDN加速域名，我参考了这篇教程：如何将 hexo 生成的博客部署至阿里云 OSS 并全站启用 CDN 加速访问。在配置的过程中，阿里云也有各种教程提示，跟着一步一步来就好。

但是关于把静态站点部署到对象存储的教程都是说 OSS 要求绝对地址，也就是 /archive/index.html 这种格式，而 /archive/ 会跳转到预定的首页，网上的解决办法都是改 Hexo 的代码给链接上加 index.html。因为感觉太麻烦了，所以一直没有改，直到看到了这篇文章：用GitHub Actions部署静态站点到OSS，才知道OSS 早就支持子目录首页，于是我也将静态站点部署到了OSS中。

之后就是利用Markdown写第一篇博客了，教程参考Markdown 教程。

2018年6月12日第一次发布

你好世界，搭一个属于自己的个人网站是大学以来一直的一个目标，然而无奈一直没有尝试实现，总觉得搭建一个网站非常非常难。但是很多事情都是值得尝试的，这一次虽然只是搭建了一个简单的WordPress博客，但是也熟悉了一下流程，以后会尝试自己写HTML网页代码，真正做一个从头到尾自己完成的网站。下面就来先分享下搭建网站的流程，作为网站第一篇文章吧。

搭建网站首先要准备一个虚拟主机和一个域名。虚拟主机的话，现如今云主机很流行，而且对学生很优惠，可选择的有阿里云，腾讯云等等。学生套餐大概一个月就只有10块钱的样子，感觉真的是很便宜了，不过腾讯云需要抢购一下，阿里云不需要（我用的是腾讯云）。不过个人博客网站的话，感觉也不需要多好的云主机，可以随意选择。

有了云主机后，还需要一个域名，否则要想访问网站时，输入的是IP地址，如果有了域名就方便多啦。域名的申请可以直接选择腾讯云或者阿里云，之后的网站备案什么的，都可以通过腾讯云来做，还是蛮方便的。

现在有了云主机和域名以后，就可以解析域名，这一步操作在腾讯云，域名管理，云解析DNS，将云主机的IP地址和域名按照要求填写好，点击确认就完成了，非常简单。

下一步就是最重要的配置网站环境了，为了图方便，我的云主机系统采用的是CentOS 6.8 64位，这样就可以根据教程来一步一步安装了（ps:其实我之前系统是Ubuntu的，自己用MobaXterm远程控制了云主机配置环境，但是后来网站出现了一点问题，就重新改了一下啦~）。我觉得根据腾讯云教程或者阿里云教程都可以很好地完成搭建个人网站。

LNMP 是 Linux、Nginx、MySQL 和 PHP 的缩写，是 WordPress 博客系统依赖的基础运行环境。我们先来准备 LNMP 环境。虽然有链接，我还是粘贴复制一下好了，步骤如下：

  1. 安装nginx

    yum install nginx -y

  2. 修改 /etc/nginx/conf.d/default.conf，去除对 IPv6 地址的监听

  3. 修改完成后，启动 Nginx：nginx，此时，可访问IP地址来确认是否已经安装成功。

  4. 将 Nginx 设置为开机自动启动：

    chkconfig nginx on

  5. 安装 MySQL：

    yum install mysql-server -y

  6. 安装完成后，启动mysql服务：

    service mysqld restart

  7. 设置mysql账号和密码：

    /usr/bin/mysqladmin -u root password ‘你的密码’

  8. 将mysql设置开机启动：

    chkconfig mysql on

  9. 安装php：

    yum install php-fpm php-mysql -y

  10. 启动php-fpm进程：

    service php-fpm start

  11. 查看php-fpm监听的端口号：

    netstat -nlpt | grep php-fpm

  12. 将php-fpm设置开机启动：

    chkconfig php-fpm on

  13. 安装并配置WordPress：

    yum install wordpress -y

  14. 配置数据库

    进入mysql（mysql -uroot -p=’你的密码’）
    创建wordpress数据库 create database wordpress
    退出:exit

  15. 把上述的 DB 配置同步到 WordPress 的配置文件中：

    vim /etc/wordpress/wp-config.php

  16. 配置 Nginx：WordPress 已经安装完毕，我们配置 Nginx 把请求转发给 PHP-FPM 来处理

    首先，重命名默认的配置文件
    cd /etc/nginx/conf.d/
    mv default.conf defaut.conf.bak
    在 /etc/nginx/conf.d创建 wordpress.conf 配置

  17. 配置后，通知 Nginx 进程重新加载：

    nginx -s reload

之后的过程就是按照自己的喜好选择WordPress主体啊，自己设计拖动一下各种按钮，来达到自己的目的。这一步就不再赘述了。

最后，一个网站还需要进行网站备案，这一步只要按照腾讯云或者阿里云的网站备案流程，一步一步填写好信息，然后等待就可以完成了。网站备案完成后，可以通过WordPress小工具，将备案图标和备案号放置到网站底部，http://www.beian.gov.cn会提供HTML代码，就完成了所有网站的工作。

完成了之后真的感觉非常的简单，所以，勇于尝试不可能吧，也许会有意想不到的收获。
